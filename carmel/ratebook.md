## Ratebooks
The main purpose of most of these tabs is to adjust trip prices (base amounts or factors) based on different trip parameters.  
For example, two tabs may both contain price adjustments. 
One tab may tie the adjustments to Zones and Car Class,  
while another may tie them to Areas and Time Slices.

> [!note]
> Other than the (Tab 1), you can double-click a row to modify the data behind it or its parameters.

### List Groups Setup (Tab 2)
This tab defines **List Price Groups** (or **List Groups**), which combine:
- Ratebook
- PU and DO location types (AP, MN, OT, etc.)
- Car Class

### Pricing Setup (Tab 1
- This tab defines the **base price of trips** based on PU and DO Zones.
- All further adjustments (from other tabs) are applied to this base value.
- Use the left section to change the **Area**.
- Use the dropdown in the top-right to change the **List Price Group**.

> [!caution]
> Do not undo the contents of this table by pressing [Esc], use the dedicated button "red x".  
> Otherwise, the tabs will stay disabled.

### Base Price Setup (Tab 3)
- This tab defines **price adjustments** (base amounts and multiplication factors).
- Adjustments are tied to PU-DO **Areas**, **Car Class**, and **List Price Group**.

### By The Hour Setup (Tab 4)
- This tab defines **hourly rates** and other parameters related to hourly pricing.

### List Price Adjustments (Tab 5)
- This tab defines **price adjustments** (base amounts).
- Adjustments are tied to **PU Zone**, **DO Zone**, and **List Price Group**.

### Airport Adjustments (Tab 6)
This tab defines price adjustments tied to:
- Airport code
- PU type (IN or OUT)
- Car Class

Tables:
- **Top table:** adjustments based on PU type
- **Bottom table:** adjustments based on Time Slice

### List Price Adjustments (Tab 5)
- This tab defines **price adjustments** (base amounts).
- Adjustments are tied to the **PU Zone**, **DO Zone**, and **List Price Group**.

### Time Slice Setup (Tab 8)
- This tab defines **Time Slices**.
- Table dates define when a Time Slice becomes active and when it ends.
- Double-click a Time Slice to edit the **daily time window** during which it is active.
